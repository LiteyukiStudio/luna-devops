# 账单与费用分析

账单页用于查看用户余额、消费概览、费用分析和账单流水。

项目空间只作为费用来源。平台会把费用扣到项目空间当前计费归属人的用户钱包；如果之后转移归属，新费用归新归属人，旧流水不迁移。

页面顶部的余额、今日花费、本月花费和待结算金额是用户账户级概览。项目空间选择位于明细工具栏，用于筛选下方的本月分类消耗、费用分析、账单流水和用量记录。未选择项目空间时，普通用户默认查看自己名下的账单流水；平台管理员默认查看全部账单。

平台管理员还可以在站点设置里配置现实货币单位和换算比例。账单页顶部概览会在 credits 后显示折算金额，例如 `1,012.24 Credits (1.01 CNY)`；这只是展示换算，底层结算仍使用 credits。

## 费用分析

费用分析按“项目空间 / 应用”聚合已结算用量，展示总费用以及构建、运行、存储、访问和其他分类费用。

如果一条历史用量没有关联到应用，页面会归到“未关联应用”。账单流水仍保留每一笔余额变动，适合审计；费用分析更适合判断哪个项目或应用正在产生主要开销。

## 访问流量

访问费用按平台访问入口的响应出站流量计费。平台不会直接读取每个 Pod 的外网流量，也不会把集群内服务互访计入公网访问费用。

网关或外部采集器需要按 GatewayRoute 和时间窗口上报响应字节数，平台按 `gateway.egress_gib` 规则换算为 credits 并写入账单。请求次数当前只保留为审计和后续防滥用分析，默认不扣费。
